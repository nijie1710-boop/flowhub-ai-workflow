# FlowHub 业务规则

## 用户层级

| 层级 | 价格 | 月调用额度 | 权限 |
|---|---|---|---|
| Free | ¥0 | 50 | 仅免费工作流 |
| Pro | ¥39/月 或 ¥372/年 | 1000 | 全场工作流 + 优先支持 |
| Team | ¥99/人/月 | 5000 | 团队协作 + API + 专属客服 |

## 自营 vs 推荐位

### 自营工作流(`type: 'self'`)
- 平台自己开发或第三方上架
- **会员可直接使用**(消耗调用次数)
- 收入来自:**订阅分账**(70% 给创作者 / 30% 平台)
- 显示绿色「自营」标签

### 推荐位(`type: 'ad'`)
- 第三方品牌购买的推广位
- **点击外跳到第三方网站**(带归因参数)
- 收入来自:**CPC / 包位费**(广告主预付)
- 显示橙色「推荐」标签

## 分账规则

### 月度结算公式
```
单个创作者收益 =
  (该创作者所有工作流当月调用次数 ÷ 平台总调用次数)
  × 当月订阅总收入
  × 70%
```

### 结算时间
- 月结
- 次月 **15 号**到账
- 最低提现门槛 **¥100**

### 示例计算
假设平台当月:
- Pro 订阅收入 ¥100,000
- 总调用 50,000 次
- PicSpark 调用 6,000 次(12%)

PicSpark 创作者本月收益 = 100,000 × 12% × 70% = **¥8,400**

## 归因机制(第三方工作流)

### URL 跳转参数
```
https://www.picspark.cn/?from=flowhub&wf_id=wf_xxx&uid=u_xxx&ts=1730000000&sig=hmac
```

### Webhook 回调
第三方在用户完成关键动作(注册 / 付费)时调用:
```
POST https://api.flowhub.cn/v1/webhook/attribution
X-FlowHub-Signature: <HMAC-SHA256>
{
  "event_type": "purchase",
  "fh_uid": "u_xxx",
  "external_uid": "pu_xxx",
  "amount": 99.00,
  "occurred_at": "2026-05-11T10:30:00Z"
}
```

### 归因窗口
- **30 天**
- 同一用户 30 天内的所有付费,都计入最初的 FlowHub 来源

## 邀请返利

- 上不封顶
- 邀请码格式:`FH` + 用户后 6 位
- 邀请链接:`https://flowhub.cn/r/<code>`

## 内容审核

### 自营工作流上架
1. 创作者提交(描述 + 示例 + 定价)
2. 平台 1-2 工作日审核
3. 通过 → `status: active`
4. 驳回 → 告知原因,可修改重提

### 推广位申请
1. 广告主选择位置 + 提交工作流信息
2. 平台审核(品类合规性 + 内容质量)
3. 通过 → 付款 → 上架
4. 驳回 → 退款

### 不允许的工作流
- 涉黄涉政
- 仿冒大牌
- 灰产工具(刷量 / 群控 / 爬虫)
- 与平台直接竞争(其他工作流市场)
